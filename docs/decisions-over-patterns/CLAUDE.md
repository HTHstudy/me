# Decisions over Patterns 문서 작성 규칙

## 목적

- 독자가 특정 상황에서 **결정을 내리는 과정**을 따라갈 수 있게 한다.
- 정답을 제시하지 않고, 판단 기준과 트레이드오프를 보여준다.

## Mental Model과의 관계

- Mental Model: "왜 이런 일이 생기는가" → 관찰과 힘의 묘사
- Decisions over Patterns: "이 상황에서 무엇을 선택할 것인가" → 결정 과정과 판단 기준
- 하나의 결정 문서에 여러 Mental Model이 교차할 수 있다.

## 형식

- 문서 상단에 제목과 질문형 인용구를 둔다.
- 인용구는 "이 선택이 무엇을 결정하는가"를 드러낸다.

## 필수 구성 요소

### 1. 선택지들
- 각 선택지마다 **"언제?"**와 **"왜?"**를 명시한다.
- 코드 예시를 포함한다.
- 각 선택의 특성과 비용을 설명한다.

### 2. 판단의 질문들
- 결정할 때 던져볼 수 있는 질문들을 나열한다.
- 질문이 어떤 선택지로 이끄는지 연결한다.

### 3. 결정 흐름 예시
- 구체적인 상황에서 어떤 질문을 던지고 어떤 선택을 하게 되는지 보여준다.
- 같은 상황에서도 조건에 따라 다른 선택이 가능함을 보여준다.

### 4. 정리하며
- 선택지별 특성을 비교하는 표를 포함할 수 있다.
- 판단의 출발점이 될 수 있는 규칙들을 정리한다.
- 규칙은 "절대적인 규칙은 아니지만, 반복적으로 유효한 패턴"으로 표현한다.

## 선택적 구성 요소

- **종류 구분**: 해당 결정에서 다루는 대상의 종류를 먼저 구분하면 결정이 명확해지는 경우
- **Side-effect 관점**: 각 선택이 미치는 영향의 범위

## 작성 관점

- 각 선택지의 장단점을 균형 있게 다룬다.
- "이것이 정답이다"가 아니라 "이런 조건에서는 이런 선택이 자연스럽다"로 표현한다.
- 왜 이 위치/방식을 선택했는지 설명할 수 있게 돕는다.

## 금지

- 특정 선택지를 "정답"으로 단정하지 않는다.
- "항상 ~해야 한다"는 표현을 피한다.
- 선택지의 비용을 생략하지 않는다.

## 위반 신호

- 하나의 선택지만 긍정적으로 서술되면 위반이다.
- "언제?" / "왜?"가 누락된 선택지가 있으면 위반이다.
- 트레이드오프 없이 장점만 나열되면 위반이다.
