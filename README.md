# Why-Driven Development

왜(Why)에 집착하는 개발자입니다.  
저는 구현보다 판단을, 패턴보다 결정을 먼저 생각합니다.  
이 문서는 개발 과정에서 내려진 선택들을 통해  
사고의 기준을 남기기 위한 기록입니다.
